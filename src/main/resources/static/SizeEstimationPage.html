I. Experiment Objective

Understand the principles of the function point method for measuring the scale of software projects. Through hands-on experiments, master the function point method. Students should work in groups based on the structure and components of the group's "Software Engineering Management and Economics" course design project to measure the scale of the project (number of function points) using the function point method. It is recommended to measure the course design project using one type of function point method and to verify the results using another function point method or another software scale measurement method. This experiment is a class design experiment project, requiring 1 hour for the experiment and 1 hour to complete the experiment report.

II. Experiment Steps

Step 1: Identify Data and Transaction Function Points

Data functions refer to usable logical data that is stored, referenced, and retrieved for updates. Data blocks and control information are logical and user-identifiable. Data functions are divided into Internal Logical Files (ILF) and External Interface Files (EIF). Transaction processing refers to operations such as external inputs, external outputs, and external queries that perform updates, retrievals, and outputs, and are divided into External Inputs (EI), External Outputs (EO), and External Queries (EQ). Please read the system design model provided in the document carefully.

Step 2: Measure Internal Logical Files (ILF)

Internal Logical Files (ILF) are user-identifiable, logically related data blocks or control information maintained within the application. ILFs are used to store data processed by one or more processes within the application. Once a data block within an application is identified as an ILF, even if it is referenced by another transaction, it cannot be considered an EIF by the same application.

Experiment Operation: Count the number of ILFs in the experimental case.

Step 3: Measure External Interface Files (EIF)

External Interface Files (EIF) are user-identifiable, logically related data blocks or control information referenced by the application under test but maintained within other applications. EIFs are used to store data referenced by one or more elementary processes within the test application. Data or control data is maintained through transactions such as additions, changes, updates, and an EIF can be referenced and calculated by multiple applications, but for one application, an EIF should only be counted once.

Experiment Operation: Count the number of EIFs in the experimental case.

Step 5: Measure External Inputs (EI)

External Inputs (EI) are the fundamental processes of the application that handle data or control information coming from outside the system's boundary. The function of an EI is to maintain one or more Internal Logical Files (ILFs) and change the behavior of the system through its processing logic.

Experiment Operation: Count the number of EIs in the experimental case.

Step 6: Measure External Outputs (EO)

External Outputs (EO) are the fundamental processes of the application that provide data or control information to the outside of its boundary. The function of an EO is to provide the user with information that has been processed beyond retrieving information or control information. The processing logic must contain at least one mathematical formula or calculation, create derived data, maintain one or more ILFs, and change the behavior of the system.

Experiment Operation: Count the number of EOs in the experimental case.

Step 7: Measure External Inquiries (EQ)

External Inquiries (EQ) are the fundamental processes of the application that provide queries of data or control information to the outside of its boundary. The function of an EQ is to provide information to the user by querying data or control information. The processing logic does not contain mathematical formulas or calculations and does not produce derived data. ILFs are not maintained during the process, and the behavior of the system is not affected.

Experiment Operation: Count the number of EQs in the experimental case.
